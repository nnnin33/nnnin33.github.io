服务器专有名词

基础硬件

实例（instance）：实例就是服务器，“创建实例”就是“开通服务器”，“重启实例”就是“重启服务器”

vCPU：1核就是1vcpu

内存(Memory / RAM)：能同时运行的程序的数量，不够就会oom

镜像 (Image)：

​ 系统镜像（system image）：只有操作系统的，如 Ubuntu、CentOS

​ 应用镜像（application image）：预装好软件了

​ 自定义镜像： 复制之前的环境配置

做实验写论文不要选应用镜像，要选系统镜像，然后选Ubuntu。因为你需要一个干净的环境来跑你自己的代码，而不是别人预装好的软件

位置、网络

地域 (Region)：服务器实体机所在的地理位置，越远延迟越高

可用区 (Zone)： “可用区A”和“可用区B”在物理上是隔离的，如果A区停电了，B区通常没事。单买一台机器时，选哪个区都一样。

公网 IP (Public IP)：只有知道这个号码，才能找到并进入这个服务器

内网 IP / 私网 IP (Private IP)：只有同一个局域网里的机器能通过这个互访。你在外网（家里/学校）连服务器时别填这个IP，连不上的

VPC (专有网络)：隔离的网络环境，服务器都在这个围墙里，外面的人进不来，除非配置公网IP和安全组

安全

安全组 (Security Group) ：如果在服务器上装了网站（80端口）或者 Jupyter Notebook（8888端口），必须去“安全组”规则里告诉保安：“允许 80 和 8888 端口的客人进来”，否则你永远打不开网页

SSH (Secure Shell)：用 VS Code 或者终端控制服务器，走的都是这个通道（默认端口是 22）

存储仓库

系统盘 (System Disk)：C盘，装操作系统的地方。一般 40G-100G 就够了

数据盘 (Data Disk)：D盘E盘，额外挂载的硬盘

快照 (Snapshot)：系统配置存档

阿里云ECS

规格族 (Family)：

  • u1 / e (通用算力型/经济型)： 便宜，适合学生、个人开发者。
  • c (计算型)： CPU很强，适合算数据。
  • g (通用型)： 也就是 CPU 和 内存 比例 1:4，比较均衡。
  • r (内存型)： 内存巨大，适合跑大型数据库。

大小 (Size)：

  • large = 2核
  • xlarge = 4核
  • 2xlarge = 8核
